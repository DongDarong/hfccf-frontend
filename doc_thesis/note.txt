Task 2 — Design Export Architecture
--------------------------------------------------

Design the export system for the Reporting module:

Requirements:

• One export service for all three report types
• Reusable export functions (PDF, Excel, Print)
• Consistent styling across all exports
• Paper size: A4 (standard)
• Orientation: Portrait (default) with landscape option for tables
• Margins: 1 inch (25.4mm) or project standard
• Header/footer: Optional (school name, date, student name)
• Page breaks: Automatic for long tables
• Print preview: Browser print dialog

Export Specifications:

PDF:
  - Format: Single-page summaries + multi-page tables
  - Styling: Preserved from web (colors, spacing)
  - Images: Embedded (student photos, badges)
  - Font: System default (no custom fonts)
  - File naming: [ReportType]-[StudentName]-[Date].pdf
  - Size limit: < 5MB per report

Excel:
  - Format: Multiple sheets (one per report section)
  - Styling: Minimal (borders, headers bold, freezed panes)
  - Data only (no images or badges)
  - File naming: [ReportType]-[StudentName]-[Date].xlsx
  - Columns: Auto-width
  - Formulas: None (values only)

Print:
  - Format: Browser print dialog
  - Styling: Print-optimized (less color, no animations)
  - Page breaks: Automatic before large tables
  - Margins: Respect browser print settings
  - Headers/footers: Optional

Task 8 — Print Stylesheet Implementation
